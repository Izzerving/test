# Полное ТЗ: AnonKeyMail (актуальная версия)

> Этот документ полностью заменяет предыдущую консолидированную версию ТЗ.
> Источник: прямое ТЗ от заказчика (входящее сообщение), приоритет выше прошлых baseline-документов.

## 1) Продукт
Создать privacy-first сервис временной почты `AnonKeyMail` (допустим ребрендинг в README при аргументации), полностью готовый к self-hosted деплою на VPS.

## 2) Критичные требования анонимности (обязательные)
1. Полный запрет на сбор пользовательских данных:
   - без cookie-трекинга;
   - без fingerprint;
   - без Google Analytics;
   - без Cloudflare tracking;
   - без любых third-party трекеров/скриптов.
2. После logout:
   - полная очистка логов пребывания и IP пользователя;
   - без исключений: удаляются все сессии пользователя и любые следы логов входа;
   - реализация строго через физическое удаление (`deleteMany`) в logout / clear-logs / purge-history;
   - `Session.keepInLogs` не используется и отсутствует в актуальной схеме.
3. Регистрация только по ключу:
   - 10–20 случайных символов (A-Z, a-z, 0-9);
   - ключ выдаётся один раз при регистрации;
   - ключ не восстанавливается и не меняется.
4. При показе ключа обязательно:
   - чекбокс «Я сохранил ключ»;
   - красный warning: потеря ключа = безвозвратная потеря аккаунта и ящиков.
5. Нет логина/email/ника/пароля — только ключ.
6. Обязательный выбор автоудаления аккаунта при регистрации:
   - 1 день / 5 дней / 10 дней / 30 дней / 90 дней / 180 дней / 1 год.
7. Обязателен надёжный cron полного удаления:
   - аккаунт + ящики + письма + история + логи.

## 3) Технический стек (строго)
- Frontend + Backend: Next.js 15 (App Router, Server Actions, TypeScript)
- UI: Tailwind CSS + shadcn/ui + dark mode default
- DB: PostgreSQL + Prisma ORM
- Realtime inbox: WebSocket (Socket.io или native) + Redis
- Mail: Postfix в Docker с catch-all + API-приём писем
- Deploy: полный `docker-compose.yml` + `.env.example` + VPS-инструкция

## 4) Тарифы и поведение

### 4.1 Free без регистрации
- 1 активный ящик одновременно.
- Пока ящик не удалён — новый создать нельзя.
- Продление: 30 минут или 3 часа.
- Домены: отдельный пул из 5 эксклюзивных free-доменов.
- Выдача адреса строго случайная, ручной выбор username/domain запрещён.

### 4.2 Free с регистрацией (по ключу)
- Базовые ограничения как в Free без регистрации.
- Продление: 30м / 3ч / 6ч / 12ч / 24ч / 48ч.
- Домены: те же 5 доменов free-пула.
- Выдача адреса строго случайная.
- В ЛК есть история ящиков.
- Активация старого ящика отключена; вместо неё CTA «Повысить тариф».

### 4.3 Premium
- До 20 активных ящиков.
- Продление: 30м / 3ч / 6ч / 12ч / 24ч / 48ч / 1 неделя / 1 месяц.
- Домены: отдельный пул из 20 эксклюзивных premium-доменов.
- Создание ящика: вручную (домен+username) или случайно.

### 4.4 Unlimited (one-time lifetime)
- До 100 активных ящиков.
- История ящиков за год + письма хранятся до ручного удаления пользователем.
- Домены: отдельный пул из 20 эксклюзивных unlimited-доменов.
- Создание: вручную или случайно.

## 5) Правила доменов (критично)
1. Пулы Free/Premium/Unlimited никогда не пересекаются.
2. Каждый домен жёстко привязан к одному тарифу.
3. Лимит на домен: максимум 500 созданных ящиков за всё время.
4. При 500 домен помечается как `exhausted` и исключается из выдачи/рандома.
5. В админке exhausted-домены маркируются и архивируются.
6. Архивные домены нельзя использовать для новых ящиков.
7. Старые ящики на архивных доменах остаются в истории и доступны к просмотру/восстановлению (если срок не истёк).
8. В Prisma-модели `Domain` обязательно:
   - `tier` enum: `FREE | PREMIUM | UNLIMITED`
   - `maxMailboxes` = 500
   - `currentMailboxes` Int @default(0)
   - `status` enum: `active | exhausted | archived`

## 6) Оплата (crypto-first, 2026)

### 6.1 Админка / Платежи
Переключатели и поля:
- Telegram Bot Token (Stars/CryptoBot)
- CryptoBot Merchant Token
- Monero: RPC URL, rpc-login, rpc-password, primary address

### 6.2 ЛК / Кнопки оплаты Premium+Unlimited
1. Telegram Stars (Bot API, `XTR`, `createInvoiceLink`)
2. CryptoBot (`create_invoice` + QR + deep-link)
3. Monero (`monero-ts`: subaddress + memo + amount + таймер 30 мин)
4. Ручной перевод (Monero/BTC адрес + memo)

### 6.3 Подтверждение платежей
- Stars/CryptoBot: webhook → auto-upgrade
- Monero: polling + `onOutputReceived` (10 confirmations) → upgrade
- Админка: таблица платежей + ручное confirm/cancel

## 7) Личный кабинет
- Уникальный ID (alphanumeric, от 5 символов)
- Текущий тариф + апгрейд-кнопки
- История ящиков (по правилам тарифа)
- Настройка автоудаления аккаунта
- Контакт с поддержкой (TG-чат из админки)
- Кнопка «Очистить логи» (без показа истории входов/сессий)
- Очистка логов/истории входов выполняется только физическим удалением сессий (`deleteMany`), без анонимизации и без `updateMany`.
- Кнопка «Удалить меня»: soft-delete (`User.deletedAt`, `Mailbox.deletedAt`), инвалидировать ключ и удалить все сессии.
- Cron каждые 5–10 минут выполняет hard-delete через 30 дней после `deletedAt` (user + cascade mailboxes/emails/payments/sessions).
