# Privacy Policy / Политика приватности

## Ключевые принципы
- сервис не использует сторонние аналитические скрипты и не отправляет пользовательские данные во внешние error-tracking сервисы;
- авторизация работает только по одноразовому ключу;
- сессионные записи не содержат IP-адреса и user-agent;
- очистка логов/сессий выполняется физическим удалением записей из базы;
- soft-delete аккаунта инвалидирует ключ, hard-delete очищает связанные сущности cron-воркером.

## Что сервис хранит
- хэш ключа доступа и lookup-хэш для быстрого поиска аккаунта;
- технические записи о почтовых ящиках, письмах, платежах и доменах;
- cookie авторизации `akm_token` (httpOnly, sameSite=strict).

## Что сервис не хранит
- email/username/password пользователя;
- IP-адреса и user-agent в сессионной таблице;
- историю входов в виде отдельного пользовательского журнала;
- сторонние аналитические/маркетинговые идентификаторы.

## Операционные логи
Сервис пишет только минимальные технические структурные логи для диагностики состояния процессов. В privacy-режиме строковые значения редактируются/хэшируются и не пересылаются во внешние системы.
